Motivation

Before YOLO being proposed, object detection networks were composed of disjoint parts, making parameters difficult to tune. Real-time detection was impossible due to multi-stage architectureâ€™s slow speed. YOLO, a one stage network, using Darknet as backbone, were proposed to solve the problems mentioned above.

Contributions

  • The author designed the YOLO model as well as Fast YOLO model and opened the source code.
  • The author combined YOLO and Fast R-CNN and got a better mAP with negligible extra time cost.
  • The author gave YOLO real-time input from webcam and validated YOLOâ€™s feasibility for real-time usage.
  • The author used YOLO for person detection in artwork and achieved great performance.

Formulations

Different from previous object detection work where neural network acted as a classifier, the author formulated objection detection as a regression problem.

A picture should be resized to 448 pixelsÃ—448 pixels first, then the picture will be divided into 7Ã—7 grid cells. Each grid cell can be described with 2Ã—5+C parameters, including two bounding boxes with coordinates and confidence information, as well as C probabilities of each class. The coordinates of each bounding box include x, y, w, h, where x and y are scaled from 0 to 1 based on the offset between the center of the bounding box and relative grid cell. The confidence of a bounding box describes IOU between ground truth bounding box and predicted bounding box if there exists one objectâ€™s center within the grid cell. The confidence of a bounding box denotes the class of the object in bounding box and the performance of bounding box.

The architecture of YOLO is shown below (Figure 1). The network was pretrained with input of 224Ã—224 resolution.

Fast YOLO has fewer convolutional layers and fewer filters than that of YOLO, so Fast YOLO has less parameters than YOLO and can achieve higher speed at the cost of losing accuracy.

The loss function was defined as follow.

Total loss is composed of localization loss and classification loss. Localization loss contains coordinate loss and confidence loss, which is related to each gird cellâ€™s first ten features. Classification loss is only calculated once in every grid cell where there exists object. The design of loss function is technical, especially the part of width and height loss, which are squared first to penalize the mistake made in small bounding boxes more.

Justification

The author compared YOLO with other object detection system and the result verified YOLOâ€™s superiority in object detection. With FPS of 45, YOLO is capable of detecting real-time objects. The author connected a webcam with YOLO and uploaded video to YouTube for demonstration.

The author analyzed error made by Fast R-CNN and YOLO, finding that YOLO is weak in localizing objects relatively while Fast-RCNN makes more mistakes in detecting background than YOLO. Hence the author combined YOLO with Fast-RCNN and achieved a better mAP.

The author used YOLO in person detection of artwork and YOLO performed perfectly compared with other algorithms. The network was trained on real photo and tested on artwork, showing that YOLO can learn general representation.

The author compared average precision on different objects, finding YOLO struggling on detecting small objects.

My thoughts

YOLOâ€™s high speed impressed me most, which is guaranteed by its unified architecture, and I think that YOLO is a milestone in real-time object detection. YOLO is more elegant compared with previous disjoint networks.

The definition of loss function is a bit tricky. The normalization of coordinates and the squaring operation are both worth learning.

The performance of combination between YOLO and Fast R-CNN is compared with other combinations between different versions of Fast R-CNNs. I think the result would be more convincing if the author tried to combine Fast R-CNN with other object detection networks even though such combinations may spend more time in detecting.

Every picture is divided into 49 grid cells, and each grid cell can predict 2 bounding boxes at most, so if there exist many small objects within one grid cell, YOLO is unable to detect. This may be one of the reasons why YOLO performs not quite well on small objects. If the picture is divided into more grid cells, it is possible for YOLO to predict small objects better, which may also interfere its speed.

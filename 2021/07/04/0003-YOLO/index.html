YOLO

Posted on 2021-07-04 Edited on 2021-07-05 In 读论文 , 暑期课程 Views:

熟悉又陌生的YOLO,借此机会读一遍。

原作者说

regression problem

以前的目标检测通过repurpose(重新调整用途)分类器来实现，YOLO把目标检测当作对bbox及其分类概率的回归问题(使用单个NN实现，可以根据检测性能端到端优化)

RPN或者DPM(deformable parts models)的pipeline慢且难于优化(各部分独立)。YOLO一次看到整张图片，Fast R CNN看的范围比较小，所以ground patch可能会被当成object。

YOLO

  • resize to 448*448
  • CNN→bboxes with class probabilities
  • threshold by confidence

confidence定义如下，反应bbox中含有object的概率，以及bbox中的obj是预测出obj的准确性:

$$
confidence\triangleq Pr(obj)*IOU^{truth}_{pred}
$$

把图片分成7×7个网格(这么说每个网格都是64px边长的正方形了)，每个网格预测2个bbox及confidence，目标中心落在哪个网格，那么这个网格就负责(responsible)检测此目标。

如果cell里没有obj，$confidence=0$，否则$confidence=IOU^{truth}_{pred}$。cell含有obj的话，预测C个类别的概率：$Pr(Class_i|Object)$。每个cell只预测一组分类概率(不管每个cell中有几个bbox)

Q:$Pr(obj)$非0即1吗？

Note：一个cell有两个bbox，cell中没有obj的话，两个bbox的$confidence=0$

一个bbox含有5个相关的值：$x,y,w,h,confidence$,$(x,y)$表示以cell为基准的box中心坐标，且经过了归一化，0~1之间取值，预测时confidence计算方式如下：
$$
Pr(Class_i|Object)*Pr(obj)*IOU_{pred}^{truth}=Pr(Class_i)*IOU_{pred}^{truth}
$$

Q:以cell左上角为原点吗？

24Conv for feature extracting + 2FC for predicting probabilities and coordinates.

前20层卷积(另外接了平均池化层+FC)首先使用ImageNet 1000分类预训练(分辨率224×224)，使用了Darknet框架。预训练完成后，使用随机权重加了4层卷积和2层FC。原因如下：

adding both convolutional and connected layers to pretrained networks can improve performance

最后一层的输出是7×7×30，包含了7×7个cell的30维特征(2个bbox的5个特征+20类的概率)。

Activation：

  • 最后一层使用linear activation function
  • 前面的都用Leaky ReLU

Loss:

包含localization和classification两部分的loss。

  • localization部分：对每个bbox，如果有负责的obj，则进行计算。
  • classification部分分为类别和概率。由于不含obj的bbox占多数，所以对于没有obj的bbox，计算loss时乘以小权重$\lambda_{noobj}$。最后的$p_i(c)$部分，每个cell只算一次、

HW2

Motivation

Before YOLO being proposed, object detection networks were composed of disjoint parts, making parameters difficult to tune. Real-time detection was impossible due to multi-stage architecture’s slow speed. YOLO, a one stage network, using Darknet as backbone, were proposed to solve the problems mentioned above.

Contributions

  • The author designed the YOLO model as well as Fast YOLO model and opened the source code.
  • The author combined YOLO and Fast R-CNN and got a better mAP with negligible extra time cost.
  • The author gave YOLO real-time input from webcam and validated YOLO’s feasibility for real-time usage.
  • The author used YOLO for person detection in artwork and achieved great performance.

Formulations

Different from previous object detection work where neural network acted as a classifier, the author formulated objection detection as a regression problem.

A picture should be resized to 448 pixels×448 pixels first, then the picture will be divided into 7×7 grid cells. Each grid cell can be described with 2×5+C parameters, including two bounding boxes with coordinates and confidence information, as well as C probabilities of each class. The coordinates of each bounding box include x, y, w, h, where x and y are scaled from 0 to 1 based on the offset between the center of the bounding box and relative grid cell. The confidence of a bounding box describes IOU between ground truth bounding box and predicted bounding box if there exists one object’s center within the grid cell. The confidence of a bounding box denotes the class of the object in bounding box and the performance of bounding box.

The architecture of YOLO is shown below (Figure 1). The network was pretrained with input of 224×224 resolution.

Fast YOLO has fewer convolutional layers and fewer filters than that of YOLO, so Fast YOLO has less parameters than YOLO and can achieve higher speed at the cost of losing accuracy.

The loss function was defined as follow.

Total loss is composed of localization loss and classification loss. Localization loss contains coordinate loss and confidence loss, which is related to each gird cell’s first ten features. Classification loss is only calculated once in every grid cell where there exists object. The design of loss function is technical, especially the part of width and height loss, which are squared first to penalize the mistake made in small bounding boxes more.

Justification

The author compared YOLO with other object detection system and the result verified YOLO’s superiority in object detection. With FPS of 45, YOLO is capable of detecting real-time objects. The author connected a webcam with YOLO and uploaded video to YouTube for demonstration.

The author analyzed error made by Fast R-CNN and YOLO, finding that YOLO is weak in localizing objects relatively while Fast-RCNN makes more mistakes in detecting background than YOLO. Hence the author combined YOLO with Fast-RCNN and achieved a better mAP.

The author used YOLO in person detection of artwork and YOLO performed perfectly compared with other algorithms. The network was trained on real photo and tested on artwork, showing that YOLO can learn general representation.

The author compared average precision on different objects, finding YOLO struggling on detecting small objects.

My thoughts

YOLO’s high speed impressed me most, which is guaranteed by its unified architecture, and I think that YOLO is a milestone in real-time object detection. YOLO is more elegant compared with previous disjoint networks.

The definition of loss function is a bit tricky. The normalization of coordinates and the squaring operation are both worth learning.

The performance of combination between YOLO and Fast R-CNN is compared with other combinations between different versions of Fast R-CNNs. I think the result would be more convincing if the author tried to combine Fast R-CNN with other object detection networks even though such combinations may spend more time in detecting.

Every picture is divided into 49 grid cells, and each grid cell can predict 2 bounding boxes at most, so if there exist many small objects within one grid cell, YOLO is unable to detect. This may be one of the reasons why YOLO performs not quite well on small objects. If the picture is divided into more grid cells, it is possible for YOLO to predict small objects better, which may also interfere its speed.

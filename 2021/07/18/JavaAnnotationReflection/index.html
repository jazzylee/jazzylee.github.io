Java 注解和反射

Posted on 2021-07-18 Views:

注解和反射是框架的底层实现机制

注解

介绍

  • 与注释的区别：注解可以被其他程序(如编译器)读取

  • 格式：@注释名(参数值)

  • 使用：附加在package/class/method/field等上面。可以通过反射机制编程实现对这些元数据的访问。

内置注解

  • @Override:重写注解

  • @Deprecated:表示不推荐使用，但是可以使用

  • @SupressWarnings():需要填写参数，如“all”/“unchecked”/等

元注解

用于注解其他注解 ，Java中定义了4个

  • @Target，描述注解使用范围
  • @Retention，描述注解的声明周期(SOURCE<CLASS<RUNTIME)
  • @Documented，说明注解包含在javadoc中
  • @Inherited，子类继承父类的注解

自定义注解

  • 其中的”方法”其实是参数
  • 返回值是参数类型，只能是Class,String,Enum
  • 只有一个参数时，一般取名为value，此时使用注解时，直接传入参数即可，不用再写value=xxx
  • 可以用default声明参数的默认值

反射⭐

概述

静态语言

运行时结构不能改变的语言，如C，C++，Java。

但反射机制可以使Java具有一定的动态性

动态语言

运行时可以改变结构的语言，如新的函数、对象，引进代码，删除已有的函数，或是其他结构上的变化。

  • Object-C
  • C#
  • JavaScript
  • PHP
  • Python

Reflection

反射机制允许程序在执行期间借助于Reflection API获得任何类的内部信息，并能直接操作任意对象的内部属性及方法。

类加载完成后，堆的方法区产生一个Class类型的对象，其中包含完整的类的结构信息。

理解Class并获取Class实例

基础理解

  • Class本身也是一个类，Class类的对象只能由系统创建
  • 一个加载的类在JVM中只有一个Class实例，对应的使一个加载到JVM中的class文件
  • 每个类的实例都会记录自己由哪个Class实例生成
  • 通过Class可以得到一个类中所有被加载的结构

进阶理解

Java内存分析

堆
  • 存放new的对象和数组
  • 所有线程共享
栈
  • 基本变量类型(包含具体数值)
  • 引用对象的变量(存放引用在堆中的具体地址)
方法区(特殊的堆)
  • 包含所有的class和static变量
  • 所有线程共享

类的加载过程

程序主动使用某个类时，如果还未加载到内存中，系统对该类的初始化步骤如下：

  • 加载，Load：读取类的class文件，创建一个java.lang.Class对象，由类加载器完成
  • 链接，Link：将类的二进制数据合并到JRE中
    • 验证：验证安全方面的问题
    • 准备：为static分配内存并初始化
    • 解析
  • 初始化，Initialize：JVM负责对类 进行初始化
    • 执行类构造器()方法，合并静态块
    • 如果发现父类还没初始化，先初始化父类

类什么时候会初始化？

  • 类的主动引用(一定会初始化)
    • JVM启动，先初始化main()所在的类
    • new出一个对象
    • 调用类的静态成员(除了final常量)和静态方法
    • 使用java.lang.reflect包的方法对类进行反射调用
    • 初始化一个类时，若其父类没有初始化，则会初始化其父类
  • 类的被动引用(不会发生初始化)
    • 访问静态域，只有真正声明这个域的类才会被初始化。如通过子类引用父类的静态变量时，不会导致子类初始化
    • 通过数组定义类引用，不会触发此类的初始化
    • 引用常量不会触发此类的初始化(常量在链接阶段就存到常量池了)

类加载器

作用

class文件加载到内存中，在堆中生成Class对象

类缓存：一旦某个类被加载到类加载器中，将维持缓存一段时间，JVM垃圾回收机制可以回收这些Class对象

分类
  • 引导类加载器：C++编写，JVM自带，负责核心库(rt.jar)的装载，无法直接获取
  • 扩展类加载器：加载jre/lib/ext目录下的jar包
  • 系统(用户)加载器

有Class对象的类型

  • class
  • interface
  • []
  • enum
  • annotation
  • void
  • primitive type(基本数据类型)

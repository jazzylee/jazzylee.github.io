Movitation

The development of object detection is shown briefly as follow(Table 1).

Method Introduction
Region-based CNN region proposal module + object/background classifier
OverFeat predicting one box‚Äôs coordinates with sharing computation
MultiBox predicting multi boxes‚Äô coordinates without sharing features
Fast R-CNN end-to-end detector training with sharing features

Region proposal plays an important role in detection. The time consumption of object detection network based on proposed regions mainly consists of region proposal time and detection time. With the method of sharing convolutions across proposals, detection time decreased a lot, thus finding a way to reduce region proposal time is an urge.

Contribution

The author replaced conventional region proposal method such as selective search with Region Proposal Network(RPN). The RPN is a fully convolutional network which use feature map generated by detect network, making it effective compared with other methods. Furthermore, the anchors are translation-invariant.

Formulation

The author experimented on ZF net and VGG-16. For ZF net, the RPN use the fifth layer‚Äôs output feature map as input (Figure 1). As to VGG-16, the same thing happens after the thirteenth layer.

In detail, the RPN consists of a 3√ó3 convolutional layer and two parallel 1√ó1 convolutional layers, one for coordinates regression while another for judging whether there is an object in the box. Loss function of RPN consists of two parts, or classification loss and regression loss. For each anchor, regression loss is zero when there is probably no object in, and such anchor is denoted as negative anchor in paper. The design of anchor is different from that of MultiBox so that the author can build pyramids of anchors compared with traditional pyramids of images or pyramid of filters.

When training the feature sharing network, the RPN part and the Fast R-CNN part are trained alternatively.

Justification

Original paper‚Äôs table 5 compares the running time. With the backbone of VGG, RPN combined with Fast R-CNN processes 10 times faster, and the same table shows that Faster R-CNN can propose quite high quality of regions. As shown in some other tables in original paper, Faster R-CNN performs better not only in speed, but also in accuracy.

My thoughts

During reading this paper, I searched Fast R-CNN and R-CNN paper, and the most impressive part in Faster R-CNN is it‚Äôs architecture. With little more calculation in RPN, Faster R-CNN achieves obvious improvement, and it‚Äôs the first time I contact with feature sharing network.

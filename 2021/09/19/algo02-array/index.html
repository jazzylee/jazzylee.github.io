数组与链表

Posted on 2021-09-19 Edited on 2021-09-20 In 散装数据结构 Views:

数组是一种线性表数据结构，用连续的内存空间存放相同类型的数据，可以随机访问。

链表是一种头皮发麻的结构，用指针将结点(零散的内存块)串在一起。

线性表

将数据拍成像线一样的结构，每个数据有前后两个方向。数组、链表、队列、栈都是这种结构。图、树、堆则是非线性表的结构，数据之间不是简单的前后关系。

低效的数据插入与删除？

插入

在长度为n的数组的第k个位置插入一个元素

  • 如果要求k~n的元素保持原有的顺序，一一后移，那么(平均)时间复杂度为O(n)
  • 如果只要求新元素在第k个位置，其他元素的顺序无所谓，那么将原来第k个位置的元素放在最后，新元素放在第k个位置，时间复杂度为O(1)----[快排中使用这种方式]

删除

在长度为n的数组中删除第k个元素，平均时间复杂度为O(n)。

如果在要删除数组中的元素时，只是标记一下该数据是删除状态，仅在内存空间不足时再一次性删除被标记的元素，元素的搬移会减少很多。

JVM的标记清除垃圾回收算法：

首先标记出所有需要被回收的对象，然后在标记完成后统一回收掉所有被标记的对象

数组与容器

容器将数组的操作封装起来， 例如动态扩容等等。

以 ArrayList为例，完全不需要关心底层的扩容逻辑，ArrayList 已经实现好了。每次存储空间不够的时候，它都会将空间自动扩容为 1.5 倍大小：

  • Java ArrayList 无法存储基本类型，比如 int、long，需要封装为 Integer、Long 类，而 Autoboxing、Unboxing 则有一定的性能消耗，所以如果特别关注性能，或者希望使用基本类型，就可以选用数组。
  • 如果数据大小事先已知，并且对数据的操作非常简单，用不到 ArrayList 提供的大部分方法，也可以直接使用数组。
  • 表示多维数组时，用数组往往会更加直观。比如 Object[][] array；而用容器的话则需要这样定义：ArrayList > array。

对于业务开发，直接使用容器就足够了，省时省力。毕竟损耗一丢丢性能，完全不会影响到系统整体的性能。但如果你是做一些非常底层的开发，比如开发网络框架，性能的优化需要做到极致，这个时候数组就会优于容器，成为首选。

MyBatis

Posted on 2021-08-05 Edited on 2021-08-10 Views:

MyBatis是一款持久层框架，支持自定义 SQL、存储过程以及高级映射。

MyBatis 免除了几乎所有的 JDBC 代码以及设置参数和获取结果集的工作。

MyBatis 可以通过简单的 XML 或注解来配置和映射原始类型、接口和 Java POJO（Plain Old Java Objects，普通老式 Java 对象）为数据库中的记录。

typeAlias

两种使用方式

  1. 为 Java 类型设置缩写名

    此后在所有需要写pojo.User的地方，都可以用User别名

  2. 指定一个包名

    MyBatis 会在包名下面搜索需要的 Java Bean，在没有注解的情况下，使用 Bean 的首字母小写的非限定类名来作为它的别名， 比如 pojo.User 的别名为 user(测试发现首字母大写也OK)。

缓存

MyBatis默认定义了两级缓存：一级缓存，二级缓存，SQL语句执行时，查找顺序：二级缓存->一级缓存->数据库.

  • 一级缓存(默认开启)

    又称本地缓存，SqlSession级别，从SqlSession建立连接到关闭连接有效。

    缓存失效的情况：

    • 增删改操作
    • 手动清理sqlSession.clearCache();

  • 二级缓存(需要手动开启)

    namespace级别。对于一级缓存，当会话提交或者会话关闭时，会转存到二级缓存中。

    启用方法：

    • 习惯上在MyBatis配置文件中显式增加配置(即使默认就是开启的)

    • 在mapper中添加<cache/>

      cache完整的配置包括：

      清除策略：默认LRU，可选下面几种：

      LRU – 最近最少使用：移除最长时间不被使用的对象。

      FIFO – 先进先出：按对象进入缓存的顺序来移除它们。

      SOFT – 软引用：基于垃圾回收器状态和软引用规则移除对象。

      WEAK – 弱引用：更积极地基于垃圾收集器状态和弱引用规则移除对象。

      刷新间隔(ms)： 默认情况不设置，也就是没有刷新间隔、缓存仅仅会在调用语句时刷新。

      引用数

      R/W配置：默认为false，只读的缓存会给所有调用者返回缓存对象的相同实例；可读写的缓存会（通过序列化）返回缓存对象的拷贝。 速度上会慢一些，但是更安全。序列化POJO就是让实体类实现Serializable接口。
